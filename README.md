# euroleague-data

Маленький отдельный репозиторий, никак не связанный с деплоем самого
Burmly. Единственная задача — раз в сутки сходить в API Евролиги и
закоммитить свежие `data/*.json` обратно в этот же репозиторий.

## Как это работает

`.github/workflows/update-data.yml` — GitHub Actions по расписанию
(каждую ночь в 03:15 UTC + можно запустить вручную кнопкой "Run workflow"
на вкладке Actions). Устанавливает `euroleague_api`, гоняет
`fetch_euroleague_data.py`, если данные изменились — коммитит.

Никакого сервера, никакой базы данных, никакого влияния на то, как
задеплоен сам сайт.

## Настройка (один раз)

1. Создай пустой репозиторий на GitHub (без README/gitignore — тут они
   уже есть)
2. Распакуй этот архив и запушь:
   ```bash
   cd euroleague-data-repo
   git remote add origin https://github.com/ТВОЙ_ЮЗЕРНЕЙМ/euroleague-data.git
   git push -u origin main
   ```
3. На вкладке **Actions** репозитория на GitHub включи workflows, если
   спросит (обычно предлагает сама после первого пуша)
4. Проверь permissions: **Settings → Actions → General → Workflow
   permissions** → должно быть выставлено **Read and write permissions**
   (без этого шаг "Commit updated data" будет падать с ошибкой push)
5. Если не хочешь ждать ночи — вкладка **Actions** → workflow
   "Update Euroleague data" → **Run workflow** (кнопка справа), запустится
   сразу

## Первый прогон — сверка полей

Реальные названия колонок в ответе API Евролиги я не проверял (нет
сетевого доступа из своей песочницы) — они основаны на документации
пакета `euroleague_api`, а не на живом ответе. Если после первого
прогона `data/teams.json` выглядит пустым или подозрительным —
запусти workflow вручную и глянь логи шага "Run data pipeline", там
`pick_col()` печатает предупреждение и реальный список колонок, если
угаданное имя не совпало. Пришли мне этот вывод — поправлю маппинг
за один проход.

## Как забрать свежие данные в Burmly

Это отдельный репозиторий специально для того, чтобы не трогать
воркфлоу деплоя самого сайта. Когда нужно обновить дашборд:

1. Зайди в `euroleague-data` репозиторий на GitHub → папка `data/` →
   скачай `teams.json`, `players.json`, `referees.json`, `shots.json`
2. Замени ими файлы в `pet-projects/sport/data/` в своей папке Burmly
3. Задеплой Burmly как обычно (drag-and-drop зип в Cloudflare)
